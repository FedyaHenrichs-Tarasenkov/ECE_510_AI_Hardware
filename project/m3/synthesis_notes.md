# M3 Synthesis Notes & Scope Status
**Date: May 23, 2026**

## Synthesis Narrative
The goal of Milestone 3 was to integrate the completely verified AXI4-Lite interface from Milestone 2 with the previously synthesized HDC Binding Unit (compute core) and push the combined `top.sv` design through the OpenLane 2 physical implementation flow. 

The integration process required minor "glue logic" to bridge the two distinct modules. Specifically, the AXI4-Lite standard mandates an active-low synchronous reset (`axi_aresetn`), while the custom HDC compute core was designed using an active-high reset (`rst`). A simple continuous assignment inverter was implemented at the top level to bridge this gap. Furthermore, the interface module from M2 had to be slightly modified to expose internal "Application-Facing" ports (`app_data_a`, `app_start`, etc.) so the parsed memory-mapped vectors could actually be routed into the compute core.

Once the RTL was integrated and verified via a co-simulation testbench (which explicitly drove the host-side AXI pins and verified the output against an independent reference), the design was synthesized.

During the initial floorplanning stages of the OpenLane flow, I anticipated a specific physical design constraint that surfaced during Codefest 7. Because the custom HDC logic is incredibly area-efficient (relying on 1-bit precision XOR gates), the OpenLane dynamic sizing algorithm (`FP_SIZING: relative`) tends to shrink the silicon die to a microscopic footprint. While this is great for area metrics, the integrated top-level module now exposes 144 external pins (the full AXI4-Lite bus). Shrinking the die too much results in a `[PPL-0024]` "pin-bound" error, where the physical perimeter of the chip is too small to place all 144 I/O pads. To circumvent this, the OpenLane `config.json` was updated to `FP_SIZING: absolute`, forcing a 250x250 µm die. This guaranteed a 1,000 µm perimeter, allowing the router to place all pins without density or spacing violations.

The flow completed successfully, achieving the target 10.0 ns (100 MHz) clock period. The integrated design utilized 960 cells (up from 132 in the standalone core) to handle the heavy AXI protocol state machines and registers. The design passed all LVS (Layout vs. Schematic) and DRC (Design Rule Checks). 

The only failure reported during the signoff stage was an Antenna Violation (`Failed 𐄂 Pin violations: 2`). In physical chip design, an antenna violation occurs when a long metal trace acts as an antenna, accumulating static charge during the plasma etching manufacturing process, which can exceed the breakdown voltage of the connected transistor's gate oxide. Because this is an automated, academic synthesis run without manual diode insertion or advanced layer promotion scripts, this violation is expected and does not impact the logical validity or the timing analysis of the RTL. Furthermore, power estimation was attempted, but the required OpenROAD voltage source configuration file (`VSRC_LOC_FILES`) was not explicitly defined in the flow, resulting in an inaccurate/skipped power report. This will be addressed directly in M4 when configuring the final benchmarking flow.

## Scope Status
The project scope remains completely unchanged. The successful synthesis of the fully integrated AXI4-Lite interface and the HDC Binding Unit proves that the 100 MHz clock target is highly achievable, even with the added routing complexity of the memory-mapped bus. The critical path analysis revealed that the bottleneck is currently the global reset fanout, not the compute logic itself. Therefore, integrating the remaining HDC modules (the bitwise accumulator Bundling Unit and the Similarity Check) for M4 remains fully viable within the established power and area constraints. The 1-bit precision architecture continues to demonstrate massive hardware efficiency compared to traditional floating-point classifiers.
